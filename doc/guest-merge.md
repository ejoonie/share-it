# Guest Account Merge

**적용일**: 2026-07-21
**관련 이슈**: #101
**PR**: #102

## 개요

게스트 계정으로 앱을 사용하다가 실제 계정으로 전환할 때, 게스트 데이터를 실제 계정으로 이전한다.

## 엔드포인트

```
POST /api/v1/my/account/merge_guest
Authorization: Bearer <실제 계정 토큰>

{ "guest_token": "<게스트 계정 토큰>" }
```

## 이전 규칙

| 데이터 | 처리 |
|---|---|
| 게스트 소유 topic (피기) | 실제 계정으로 소유권 이전 (기존 유저면 topic이 추가됨) |
| 게스트 topic의 `is_default` | 기존 유저면 `false`로 변경 (default topic 중복 방지) |
| 샘플 entry | 삭제 |
| entry `created_by` / `updated_by` | 실제 계정으로 이전 |
| 내가 팔로우하던 topic | 실제 계정으로 이전 (충돌 시 기존 것 유지) |
| 나를 팔로우하던 구독 | topic과 함께 자연스럽게 유지 |
| 게스트 계정 | 삭제 |

> 모든 작업은 트랜잭션으로 묶여 있어 중간 실패 시 롤백된다.

## 시나리오

### 시나리오 1 — 신규 유저 (게스트 → 첫 실제 계정)

1. 게스트로 앱 실행 → 피기 생성, 지출 입력
2. 이메일로 회원가입
3. `merge_guest` 호출
4. **기대**: 게스트의 피기와 지출이 새 계정에 그대로 있음, `is_default` 유지

### 시나리오 2 — 기존 유저 (게스트 → 이미 계정 있음)

1. 게스트로 앱 실행 → 피기 생성, 지출 입력
2. 기존 계정으로 로그인
3. `merge_guest` 호출
4. **기대**: 게스트 피기가 기존 피기에 추가됨, 게스트 피기의 `is_default`는 `false`로 변경

### 시나리오 3 — follow 이전

1. 게스트로 남의 피기를 팔로우
2. 실제 계정으로 로그인 후 `merge_guest` 호출
3. **기대**: 게스트가 팔로우하던 피기가 실제 계정에 유지됨

### 시나리오 4 — 팔로워 유지

1. 다른 유저가 게스트의 피기를 팔로우 중
2. 게스트가 실제 계정으로 merge
3. **기대**: 팔로워의 구독이 이전된 피기에 그대로 유지됨

### 시나리오 5 — 기존 유저가 이미 같은 피기를 팔로우 중

1. 게스트와 실제 계정 둘 다 같은 topic을 팔로우 중
2. `merge_guest` 호출
3. **기대**: 실제 계정의 기존 팔로우 유지, 게스트 팔로우는 스킵

### 시나리오 6 — 게스트에 데이터 없음

1. 게스트로 앱 실행 (아무것도 입력 안 함)
2. 로그인 후 `merge_guest` 호출
3. **기대**: 200 성공, 게스트 계정만 삭제

### 시나리오 7 — 같은 guest_token으로 두 번 호출

1. `merge_guest` 호출 → 성공 (게스트 삭제됨)
2. 동일 `guest_token`으로 재호출
3. **기대**: 404 반환

### 시나리오 8 — 트랜잭션 실패

1. 이전 도중 DB 오류 발생
2. **기대**: 변경사항 전체 롤백, 게스트 계정 유지
